# Changelog

Усі помітні зміни в цьому проєкті будуть документуватися в цьому файлі.
Формат базується на [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
і цей проєкт дотримується [Семантичного версіонування](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2026-06-17

### Added
- **Базова бізнес-логіка (ЛР №3):** Початкова реалізація сутностей предметної області — класів `Book`, `User` та рольової моделі користувачів.
- **Архітектурні патерни (ЛР №4):** Інтегровано паттерн `Singleton` для контролю унікальності об'єктів у пам'яті та паттерн `Observer` для реалізації системи динамічних сповіщень читачів.
- **Модульна декомпозиція (ЛР №5):** Розділено єдину кодову базу на ізольовані модулі: `book.py`, `user.py`, `library.py`, `notifications.py` та виділено слабкі зв'язки через інтерфейси в `interfaces.py`. Створено головний сценарій `main.py`.
- **Автоматизоване тестування (ЛР №6):** Розроблено тестовий пакет із 17 базових модульних тестів на базі фреймворку `unittest` з використанням `mock`-об'єктів для підміни залежностей.
- **Обробка винятків (ЛР №6 - Рефакторинг):** Додано бізнес-валідацію в конструктор книги (`Book`) та покриття негативного сценарію через контекстний менеджер `assertRaises(ValueError)`.
- **Тестування точок входу (ЛР №6 - Підвищений рівень):** Створено окремий модуль `test_main.py` для повноцінного інтеграційного тестування головного сценарію додатка з перехопленням консольного виводу `sys.stdout`.
- **Конфігурація VCS (ЛР №7):** Створено файли контролю конфігурації `.gitignore`, опису проєкту `README.md` та ліцензії `LICENSE`.

### Changed
- **Рефакторинг тестів (ЛР №6):** Усунено дублювання коду (код-запах *Duplicate Code*) у модулі `test_user.py` шляхом винесення спільних налаштувань користувачів та Mock-фікстур у метод `setUp()`.
